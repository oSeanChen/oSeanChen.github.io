Rails 套件操作指南，Devise (安裝及基本設置)

Created2022-08-27|Updated2022-08-27|Rails套件
|Post View:

進入 ruby 世界之後，無非就是希望可以使用 rails 框架來幫助你達到快速及 RESTful 的開發方式，在 rails 世界其實擁有許多強力的裝備，可以幫你擊退沿路的困難，就是我們所謂的套件 (packages)。

想當初學習自製登入系統真的有點辛苦呢，但 rails 中其實有厲害套件 Devise 可以幫你完成喔，但要能客製化出自己想要的東西也不是很容易的事情呢! 就先來針對基本的使用及客製方式好好來了解一下吧。

安裝

原始 Devise github

  1. 放到 gemfile 然後 bundle 進行安裝
  1. 產生 devise 設定檔

基本設置

  1. 設定寄信的 mailer 預設 url 位置
    • 開發階段 development：
      設置為 localhose:3000 的位置
      config/environments/development.rb
    • 上線階段 production：
      如果上線後可以將位置改為實際部屬位置
  2. 設置首頁位置 routes
    加入 root 確定登入或是相關操作跳轉回去的位置
    config/routes.rb
    root to: "home#index"
  3. 通知訊息設定，notice 及 alert
    預設已經寫好很多 notice 及 alert，application.htnl.erb 中有加入就可以看到。
    app/views/layouts/application.html.erb.

建立 model

透過內建指令產生 model 及 migration，MODEL(通常用 User)，可以用別的但會影響預設的 controller 跟 view 位置及名稱，可以到 db/migrate 查看產生的 migration 檔案

產生後要記得 db:migrate

產生預設相關 view

使用這個指令可以產生所以套件已經建立的 view，
可以調整需要的頁面連結，註冊、登入、忘記密碼等等，客製自己的版面

  • sessions 登入相關
  • registrations 註冊相關
  • confirmations 帳號認證相關
  • passwords 忘記密碼/密碼修改
  • unlocks 上鎖帳號相關
  • mailer 信件寄送內容相關
  • shared 共同
    • error_messages 錯誤訊息顯示
    • link 所有連結彙整 (登入、登出、第三方登入註冊、忘記密碼…)

做到這邊就可以看到登入系統已經有可以使用基本的功能及畫面了
根據相關的路徑可以找到對應的畫面

/users/sign_up 註冊

/users/sign_in 登入
